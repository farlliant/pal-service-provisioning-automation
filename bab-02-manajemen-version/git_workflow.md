# Contoh Alur Kerja Git Sederhana

## Skenario
Seorang anggota tim ingin menambahkan fitur login.

## Alur
1. Ambil versi terbaru dari branch utama.
2. Buat branch baru, misalnya `feature/login`.
3. Kerjakan perubahan secara bertahap.
4. Commit kecil dengan pesan yang jelas.
5. Push branch ke remote repository.
6. Buat pull request.
7. Setelah direview, lakukan merge ke branch utama.

## Contoh Nama Branch
- `feature/login`
- `feature/dashboard-admin`
- `bugfix/navbar-mobile`
- `docs/update-readme`

## Tujuan Alur Ini
- menjaga branch utama tetap stabil,
- memudahkan review,
- memudahkan rollback,
- memperkecil konflik.
